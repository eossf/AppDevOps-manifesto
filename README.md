### Principles behind the Feature Craft Manifesto

1. **Absolute priority**: deliver real user value in the app, as fast as possible – via AI Feature Craft that transforms features into functional code in minutes not weeks.

2. **Ultra-rapid delivery**: deploy AI-crafted features **multiple times per day** – relying on CI/CD and DevOps principles mature for over a decade.

3. **Co-authors of the Book**: daily work together on a shared artifact (the "Feature Book") – PM writes in natural language, Dev adds technical and security context, AI crafts the code. Showing results as fast as possible.

4. **AI as an omnipresent co-team-member**: Specialized agents, local skills/tools, multi-agent memory, connected IDEs, specification frameworks – all available today. This is the AI ecosystem.

5. **Mastered enterprise context**: every app relies on an internal "LLM Forge" (like Mistral Forge) – proprietary data, business vocabulary, compliance constraints, specific workflows – for reliable, aligned, and sovereign agents (privacy, no leakage).

6. **Sustainable pace**: AI crafts the code → humans (PM + Dev) focus on creativity, strategy, edge cases, architecture

7. **Technical excellence + AI**: good design, clean code, scalability emerge from collaboration + Dev guidance on AI craft – no tech debt despite the speed.

8. **Radical simplicity**: maximize work not done – PM specifies outcomes and user value, AI + Dev craft the rest. Less unnecessary technical specs.

9. **Self-organizing team and agents around the App**: shared ownership PM-Dev-AI; the best "Feature Book" emerges from a dedicated team when agents are involved according to needs, all along the product lifecycle. 

10. **Continuous improvement**: regular retrospectives on Feature Craft effectiveness, the shared Book, and contextual AI – adjust agents, prompts, internal Forge.
